# 形式审查 & 内容完整性清单

评审前先逐项检查以下清单。标记结果：✅ 通过 | ❌ 未通过 | ⚠️ 缺失 | ➖ 不适用

---

## A. 基本格式要求

- [ ] **A1** 文件格式为 PDF，可正常打开和阅读
- [ ] **A2** 有封面页，包含项目名称、团队名称、院校、日期
- [ ] **A3** 有目录或大纲，章节编号清晰
- [ ] **A4** 页码完整连续
- [ ] **A5** 全文字体、字号、行距基本统一，排版整洁
- [ ] **A6** 图表有编号和标题，分辨率可读

## B. 核心内容完整性

- [ ] **B1** 项目摘要/概述（300 字以内概括项目核心）
- [ ] **B2** 问题背景与痛点分析（要解决什么问题，为什么重要）
- [ ] **B3** 解决方案描述（产品/服务是什么，怎么解决问题）
- [ ] **B4** 创新点阐述（与现有方案的差异化）
- [ ] **B5** 商业模式（谁付费、怎么赚钱、定价策略）
- [ ] **B6** 市场分析（市场规模、目标用户画像、增长趋势）
- [ ] **B7** 竞品分析（至少分析 2-3 个竞争对手）
- [ ] **B8** 营销/推广策略
- [ ] **B9** 财务规划（成本结构、收入预测、盈亏平衡点）
- [ ] **B10** 团队介绍（成员背景、分工、指导老师）
- [ ] **B11** 发展规划/里程碑（短期、中期、长期目标）
- [ ] **B12** 风险分析与应对措施

## C. 数据与论据质量

- [ ] **C1** 市场数据标注了来源（报告名称、机构、年份）
- [ ] **C2** 财务数据的核心假设有明确说明
- [ ] **C3** 用户调研/访谈数据（如有）说明了样本量和方法
- [ ] **C4** 无明显数据造假或自相矛盾
- [ ] **C5** 引用的政策/法规信息准确且时效性合理

## D. 加分项（有则加分，无不扣分）

- [ ] **D1** 已有原型/MVP 展示（截图、链接、演示）
- [ ] **D2** 已获得用户/客户/合作方的实际反馈
- [ ] **D3** 已有收入或签约意向
- [ ] **D4** 已申请/获得专利、软著、商标等知识产权
- [ ] **D5** 获得过相关比赛奖项或孵化器入选
- [ ] **D6** 有指导老师推荐信或专家背书

## 审查结果汇总模板

```
形式审查通过率：X / Y 项（必查项）
加分项命中：X / 6 项

关键缺失项：
- BX: ...（影响评估：高/中/低）
- ...

形式审查结论：通过 / 有条件通过 / 不通过
```

### 判定标准
- **通过**：A 类全部通过，B 类至少 10/12 通过，C 类无重大问题
- **有条件通过**：有 1-2 项 B 类缺失但不影响核心评审
- **不通过**：A 类有未通过项，或 B 类缺失 3 项以上核心内容
