# 正大杯项目评审助手

## 项目概述

本项目用于辅助「正大杯」大学生创新创业大赛的参赛作品评审。  
Claude 将扮演**资深评审专家**，对 `inputs/` 中的参赛材料（PDF）进行结构化评审，按照 `rules/` 中的评分标准和检查清单逐项打分，并将评审报告输出到 `outputs/`。

## 目录约定

| 路径 | 用途 |
|---|---|
| `inputs/` | 放入待评审的 PDF 文件（计划书、PPT导出、附件等） |
| `outputs/` | 评审结果自动输出于此，命名格式 `{原文件名}_评审报告.md` |
| `rules/review-prompt.md` | 评审角色与整体提示词 |
| `rules/scoring.md` | 评分维度、权重与评分标准 |
| `rules/checklist.md` | 形式审查 & 内容完整性清单 |
| `scripts/review.sh` | 单文件评审脚本 |
| `scripts/batch-review.sh` | 批量评审脚本 |

## 核心工作流

1. **读取规则**：每次评审前先完整阅读 `rules/` 下三个文件，理解评分框架。
2. **读取材料**：读取 `inputs/` 中指定的 PDF 文件。
3. **结构化评审**：
   - 先执行 `checklist.md` 中的形式审查，输出通过/未通过。
   - 再按 `scoring.md` 中的 5 个维度逐项评分（1-10 分），给出得分理由。
   - 最后给出加权总分、核心优势、关键不足、改进建议。
4. **输出报告**：写入 `outputs/` 目录，格式为 Markdown。

## 评审原则

- **严格对标评分标准**，不凭印象给分。
- 每个扣分点必须**引用原文具体位置**说明问题。
- 改进建议必须**具体可操作**，不说空话。
- 如材料缺失关键部分，在报告中标记为「⚠️ 缺失」并说明影响。
- 保持客观中立，对所有作品使用相同标准。

## 快速开始

```bash
# 评审单个文件
bash scripts/review.sh inputs/项目计划书.pdf

# 批量评审 inputs/ 下所有 PDF
bash scripts/batch-review.sh
```

## 注意事项

- 输入文件仅支持 PDF 格式。
- 评审报告会自动带上时间戳，不会覆盖已有报告。
- 如需自定义评分维度或权重，直接修改 `rules/scoring.md`。
